🔐 Sessions in Checkout Page - Technical Documentation

📋 What are Sessions?

Sessions are a way to store user data temporarily on the server-side (backend) while a user is browsing your website. Unlike localStorage (client-side), sessions are managed by the server and are more secure for sensitive operations like authentication and checkout.

🔄 How Sessions Work in Your Checkout Page

Your application uses dual session management:

  1. Server-side sessions (Express.js with express-session)
  2. Client-side localStorage for UI state

1. 🖥️ Server-Side Session Setup (server.js)

// Session middleware configuration app.use(session({ secret: process.env.SESSION_SECRET || 'dev_session_secret', resave: false, saveUninitialized: false, cookie: { maxAge: 24 * 60 * 60 * 1000, // 24 hours httpOnly: true, // Prevents XSS attacks sameSite: 'lax', secure: false, // false for localhost domain: 'localhost' } }));

Purpose: Creates secure server-side storage for user authentication data.

2. 🔑 Login Creates Session (routes/profile_register.js)

// After successful login req.session.user = { id: userRow.id, name: userRow.name, email: userRow.email, userType }; req.session.save((err) => { if (err) { console.error("❌ Session save error:", err); return res.status(500).json({ success: false, message: "Session save failed" }); } return res.json({ success: true, message: "Login successful", user }); });

Purpose: Stores authenticated user data on the server.

3. 🛒 Checkout Page Uses Sessions (checkout.html)

function ensureBuyerLoggedIn() { const isLoggedIn = localStorage.getItem('isLoggedIn') === 'true'; const userSessionJSON = localStorage.getItem('userSession'); if (!isLoggedIn || !userSessionJSON) { showLoginRequired('You must be logged in as a buyer to proceed to checkout.'); return false; } try { const userSession = JSON.parse(userSessionJSON); if (userSession.userType !== 'buyer') { showLoginRequired('Please login as a buyer to complete the checkout process.'); return false; } return true; } catch (e) { showLoginRequired('Please login to continue.'); return false; } }

Purpose: Validates user authentication before allowing checkout.

4. 💳 Checkout Process (routes/cart.js)

router.post('/checkout', (req, res) => { const { cart_key, customer_name, email, address, payment_method } = req.body; // No direct session check here - relies on frontend validation // But server-side sessions could be used for additional security // Process order... });

🎯 Key Uses of Sessions in Checkout

  • Authentication: Ensures only logged-in buyers can checkout
  • User Data: Stores user profile information
  • Security: Server-side storage prevents tampering
  • State Management: Maintains login state across page refreshes

🛡️ Security Benefits

Security Features:

  • HTTPOnly cookies: Prevent XSS attacks
  • Server-side storage: Data not accessible via JavaScript
  • Expiration: Sessions auto-expire (24 hours in your config)
  • Secure flags: Can be enabled for HTTPS in production

📋 Session Management Functions

Check Current Session (routes/profile_register.js)

// Return current session user router.get('/me', (req, res) => { if (req.session && req.session.user) { return res.json({ success: true, user: req.session.user }); } return res.json({ success: false, user: null }); });

Logout - Destroy Session (routes/profile_register.js)

// Logout - destroy session router.post('/logout', (req, res) => { if (req.session) { req.session.destroy(err => { if (err) { console.error('Error destroying session:', err); return res.status(500).json({ success: false, message: 'Logout failed' }); } return res.json({ success: true, message: 'Logged out successfully' }); }); } else { return res.json({ success: true, message: 'No active session' }); } });

⚠️ Important Notes

Production Considerations:

  • Set secure: true for HTTPS environments
  • Use a strong, unique SESSION_SECRET
  • Consider using Redis or database for session storage in production
  • Implement session rotation for security

🔍 Debugging Sessions

To debug session issues in your application:

// Check server-side session console.log('Server session:', req.session); // Check client-side localStorage console.log('Client session:', localStorage.getItem('userSession')); console.log('Is logged in:', localStorage.getItem('isLoggedIn'));
